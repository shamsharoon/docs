## Overview

Retrieve all stations on a specific GO Transit route. This endpoint provides detailed information about a route including all station stops, their sequences, and direction information.

## Parameters

<ParamField query="route" type="string" required>
  Route name or identifier. Can be the full route name or a partial match.
  
  Examples: `Lakeshore East`, `Kitchener`, `LSE`, `Kit`
</ParamField>

## Response Fields

<ResponseField name="route" type="string">
  The route name as provided in the request.
</ResponseField>

<ResponseField name="stations" type="array">
  Array of stations on the route with the following structure:
  
  <Expandable title="Station Object">
    <ResponseField name="stop_id" type="string">
      Unique stop identifier from GTFS data.
    </ResponseField>
    
    <ResponseField name="stop_name" type="string">
      Full station name.
    </ResponseField>
    
    <ResponseField name="stop_sequence" type="integer">
      Position of the station on the route (1-based).
    </ResponseField>
    
    <ResponseField name="direction" type="string">
      Direction of travel for this station sequence.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="count" type="integer">
  Total number of stations on the route.
</ResponseField>

<ResponseField name="attribution" type="string">
  Data attribution notice from Metrolinx.
</ResponseField>

## Use Cases

### Building Station Lists

Use this endpoint to populate dropdown menus or station selection interfaces in your applications.

### Route Planning

Combine with the `/next` endpoint to help users select valid origin and destination stations.

### Station Discovery

Help users discover all available stations on a particular route they're interested in.

## Tips

<Tip>
Route names are case-insensitive and support partial matching. You can use short codes like "LSE" or full names like "Lakeshore East".
</Tip>

<Tip>
The `stop_sequence` field shows the order of stations along the route, which can be useful for visualizing route paths.
</Tip>

<Tip>
Use this endpoint in combination with `/routes` to first discover available routes, then get their station details.
</Tip>
